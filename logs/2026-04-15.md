# 2026-04-15

## Dashboard → Log deep-link for "Due today" routines

When a scheduled routine is due today, the `ScheduleNotice` widget on the
Dashboard now renders each routine as a tappable chip that jumps straight
into logging it — no need to open Log → Train → Routines and hunt for the
right one.

### Changes

- **betalog-react/src/pages/Dashboard.jsx**
  - `ScheduleNotice` now pulls `useRoutines` + `useHangRoutines` so it can
    resolve each scheduled `routineId` to its kind (gym vs hang).
  - "Due today" line replaced with a flex-wrap row of coloured chips
    (blue for gym routines, purple for hang routines), each a button that
    calls `navigate('/log', { state: { openRoutine: { id, kind } } })`.
  - Disabled state for chips whose routine no longer exists.

- **betalog-react/src/pages/Log.jsx**
  - Reads `location.state.openRoutine` on mount via `useEffect`.
  - For `kind: 'gym'` → sets mode=train, tab=routines, opens GymLogSheet
    with `{ type: 'routine', id }`.
  - For `kind: 'hang'` → sets mode=hang, opens HangboardTimer with the
    resolved hang routine object.
  - Clears router state after handling so tab switches don't re-trigger.

### Verified

- `npm run build` — clean, 1.23s, no new warnings.

## Fix: Log page blank-screen after deep-link

Calling `navigate()` from inside the `useEffect` that consumed
`location.state` caused a render loop / crash. Replaced with a
`useRef` one-shot guard keyed on `kind + ':' + id` so the request
is handled exactly once per unique deep-link without touching the
router mid-effect.

## Weight logging: decouple from widget-toggle save

**Bug:** ProfileTab bundled weight saving and dashboard-widget
toggles under a single "Save" button. Toggling a widget saved the
profile and also re-logged today's weight using the *stale*
last value, polluting history with fake daily entries.

### Changes — betalog-react/src/components/profile/ProfileTab.jsx

- Removed bundled Save button.
- Dashboard widget toggles now persist immediately on click via
  `saveProfile({ dashWidgets: next })`.
- Weight has its own inline `Log` button that only activates when
  the typed value differs from the last logged entry.
- Weight hint is now phrasal: "logged today", "logged yesterday",
  "logged Nd ago", or a short date beyond 7 days.
- Pressing Log upserts today's weight entry and updates
  `profile.weightKg` — nothing else is touched.

## GymLogSheet: tick off exercises during a routine

When logging a multi-exercise routine (e.g. Ankle physio with 4
exercises) it was hard to tell where you were. Each exercise card
now has a tap-target done indicator — only for routine sources,
single-exercise logging is unchanged.

### Changes — betalog-react/src/components/log/GymLogSheet.jsx

- `ExerciseCard` accepts `onToggleDone`. When provided, header
  shows a round checkbox; tapping it toggles `card.done`.
- Done cards render with green border, 60% opacity, and strike-
  through on the name.
- `toggleDone(cardIdx)` added to GymLogSheet.
- Routine cards built with `done: false`.
- Sheet header shows `doneCount/totalCount done` next to the
  date (blue while in-progress, green when complete).
- `done` is ephemeral UI state — `handleSave` picks only specific
  session fields so it never leaks into persisted sessions.

## Persist skipped routine exercises into history + coach context

Extended the tick-off feature so skipped exercises are preserved in
the saved session. Makes partial completions visible in the History
feed and in the AI coach context.

### Data model — betalog-react/src/lib/types.js

- Added optional `done?: boolean` to `SessionExercise`.
- Legacy sessions (no `done` field) are treated as done. Only
  explicit `done === false` is surfaced as "skipped".

### Save path — betalog-react/src/components/log/GymLogSheet.jsx

- `handleSave` now includes `done` on each SessionExercise.
  Non-routine (single-exercise) logs always save `done: true`.
  Routine logs save `!!card.done`.
- Edit mode prefills each card's `done` from the saved session
  (defaults to `true` for legacy sessions).

### History feed — betalog-react/src/components/log/SessionCard.jsx

- New `routineCompletion()` helper returns `{done, total}` only
  when a routine session has at least one skip.
- Card now shows an orange `N/M` pill next to the difficulty badge
  when a routine was partially completed.
- `gymDetail` summary prefers the done exercises in the truncated
  preview line so the reader sees what was actually done.

### Detail sheet — betalog-react/src/components/log/SessionDetailSheet.jsx

- Skipped exercises render with orange border, 60% opacity,
  strike-through name, and an explicit `SKIPPED` pill.

### AI coach context — betalog-react/src/pages/Coach.jsx

- `buildContext` marks skipped exercises as `(SKIPPED)` in the
  session log.
- For routine sessions with partial completion, adds a
  `completed N/M` line so the model can reason about adherence.

### Verified

- `npm run build` — clean.
