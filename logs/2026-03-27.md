## What we did

### Firebase integration (Step 7)
- Firebase project `betalog-340b3` fully configured (auth, Firestore, security rules)
- Installed Firebase SDK (`npm install firebase`)
- Created `src/lib/firebase.js` — config file with project credentials
- **Auth:**
  - Login screen with Google sign-in + email/password option
  - Email: sign in / create account toggle, friendly error messages
  - Auth state listener in App.jsx (loading → login → app)
  - Sign out button + user info (avatar, name, email) in Settings
- **Firestore sync:**
  - `Storage.syncToFirestore(userId)` — writes all syncable data to `users/{userId}` doc
  - `Storage.pullFromFirestore(userId)` — reads cloud data on login
  - `Storage.mergeFromCloud(cloudData)` — writes cloud data into localStorage
  - `setDataAndSync` wrapper — every `setData` call also triggers background Firestore sync
  - On first login with existing localStorage data: pushes local → cloud
  - On login to new device (empty localStorage): pulls cloud → local
  - "Syncing…" toast while pulling from Firestore
- **Security:** Firestore rules lock each user to their own `users/{userId}` path

### Friends feature (Step 8)
- **Friend codes:** Each user gets a unique `BL-XXXXX` code stored in Firestore + `friendCodes/{code}` index for fast lookups
- **Add/remove friends:** Bidirectional — entering a code links both users' `friends` arrays
- **Public profile:** `users/{userId}/public/profile` doc written on every sync, contains: displayName, boulder/rope levels, streak, last 5 sessions (summary only)
- **Friends sheet:** Slide-up sheet (matches Settings pattern), accessed via Users icon in header bar next to settings cog
  - Your code section with copy button
  - Add friend input (validates format, checks duplicates, handles own-code)
  - Friends list with cards showing: avatar initial, name, streak, boulder/rope levels with badges, recent sessions, "updated X ago" timestamp
  - Remove friend with tap-to-confirm pattern
- **Security:** Firestore rules updated — public profile only readable by friends (not all authenticated users), friend code index readable by any authenticated user
- **Refactor:** Extracted pure data functions (grade stats, streak calc, level mapping) from `ClimbingStats.jsx` and `Dashboard.jsx` into `src/lib/stats.js` to avoid circular dependencies with storage.js
