## Session 2: exercise library expansion + iOS UX polish

### Climbing grip exercises — new library additions

**Why:** App is for climbers — the exercise library had no finger/grip strength work despite this being the most sport-specific training area. Block pulls (crimp/open hand) are particularly trending right now as a structured max-strength method.

**Duplicate purge (prerequisite):**
- Added `RETIRED_EXERCISE_IDS = ['e72','e73','e77','e78','e79']` export to `defaultExercises.js`
- Updated `seedDefaultExercises` in `useExercises.js` to filter retired IDs out of localStorage on load — stale duplicates (Bicep Curls cable, Chest Press machine etc.) removed automatically for existing users
- Purge + new-defaults add happen in a single write; idempotent on repeat loads

**12 new grip/climbing exercises (e90–e101), category `arms`:**
- Placed under `arms` not a separate `climbing` category — anatomically these are all forearm/finger work; keeping taxonomy clean and anatomically grounded
- Pinch Block Hold (isometric, other, time) — most neglected grip type for climbers
- Plate Pinch (isometric, other, time) — gym-accessible pinch version
- Rice Bucket (isometric, other, time) — rehab + dual flexor/extensor conditioning
- Towel Pull-ups (pull, bw, reps) — grip-intensive pulling; high transfer to rock texture
- Finger Extensions (push, band, reps) — theraband antagonist; #1 injury prevention for climbers
- One Arm Dead Hang (isometric, bw, time) — advanced unilateral test + strength builder
- Finger Rolls (pull, db, reps) — direct finger flexor tendon work; bar held on fingertips only
- Wrist Pronation/Supination (isometric, db, reps) — prevents medial epicondylitis (climber's elbow)
- Crimp Block Pull (pull, other, reps) — max strength; crimp block on loading pin, deadlift from floor
- Open Hand Block Pull (pull, other, reps) — same but safer grip position; tendon health priority
- Two Finger Block Pull (pull, other, reps) — pocket simulation; middle+ring pair
- Weighted Block Hang (isometric, other, time) — added-weight hang from block; max strength via time-under-tension

### storage.js — migrateRoutine bug fix

**Bug:** `Storage.load()` called `migrateRoutine(r, 'gym')` on all `il_routines`, forcibly overriding every routine's `type` to `'gym'`. This meant seeded hangboard routines (type: `'hangboard'`) were immediately promoted to gym routines on load — appearing in the gym routine list with an exercise picker rather than the hang timer.

**Fix:** Removed the forced `'gym'` argument for `il_routines`. The forced type is now only applied to old `il_hbRoutines` (legacy key being migrated). `migrateRoutine` signature: `function migrateRoutine(r, forcedType)` — `forcedType` is now optional; if absent, `r.type` is preserved.

## Session 10: iOS safe area fixes

### Problem

On iPhone (notch/home indicator models), three issues:
1. Bottom nav tabs were slightly obscured by the home indicator bar
2. Bottom nav tabs were clipped left/right by the device's safe area insets
3. The progress bar in HangboardTimer rendered near the middle of the screen rather than at the true bottom — and the safe-area padding wasn't applying at all

**Root cause:** `env(safe-area-inset-*)` was returning `0` everywhere because `viewport-fit=cover` was missing from the meta viewport tag. Without it, the browser doesn't expose safe area values.

### Fixes

- `betalog-react/index.html` — added `viewport-fit=cover` to meta viewport:
  ```html
  <meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
  ```
- `Nav.jsx` bottom nav — added safe area padding to the nav `style`:
  ```js
  paddingBottom: 'env(safe-area-inset-bottom)',
  paddingLeft:   'env(safe-area-inset-left)',
  paddingRight:  'env(safe-area-inset-right)',
  ```
- `HangboardTimer.jsx` main container — added `paddingBottom/Left/Right: env(safe-area-inset-*)` so the progress bar sits flush with the true bottom of the usable screen area
